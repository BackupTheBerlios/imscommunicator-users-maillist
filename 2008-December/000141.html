Dear Franz,
I've tested with the latest's releases of ims-communicator (r97) and OpenIMSCore (r608), and the IMS-Communicator successfully registers using UDP or TCP.
For TCP, please change the following properties in ims-communicator.xml:
- <net.java.sip.communicator.sip.REGISTRAR_TRANSPORT value="TCP">
- <net.java.sip.communicator.sip.TRANSPORT value="TCP">
- <javax.sip.OUTBOUND_PROXY value="host:port/tcp">

Let me know if you keep having this issue.

BR,
Miguel Freitas

Hi,

I changed the transport parameter in ims-communicator.xml
(REGISTRAR_TRANSPORT and OUTBOUND_PROXY) to TCP and found that now IMS
communicator sends registration directly to I-CSCF. It ignores the P-CSCF
setting. I guess that it now used DNS to find the next hop.

The registration in this case does not work (as can be expected). I-CSCF
rejects the REGISTER with "400 Bad Request - P-Visited-Network-ID header
missing".
