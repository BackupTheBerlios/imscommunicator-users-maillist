Hi Johannes,
 
> i am currently experimenting with OpenIMSCore and the IMS-Communicator.
> When I try to register, it fails with "Forbidden - HSS User Unknown in
> this context".
> Registration works fine with OpenIC Lite, and if I look at the wireshark
> trace I don't see any big difference between the packets sent by the two
> clients. 

Yes but there is a difference. Since r570 of OpenIMSCore the registration
has been enhanced by using the qop-parameter in WWW-Authenticate header
field. This requires the client to also set the qop-parameter (and adapt the
security algorithm accordingly). This is done by OpenIC Lite but not by IMS
Communicator.

Be aware that 
- Session setup with OpenIC Light (v1.2) has also an issue: No ACK is sent
- There is a patch to disable the "qop" mechanism.
