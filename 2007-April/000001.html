Hi there...
About the first question, what are you trying to do? Do you want to send only the Register request with TCP and other requests using UDP? I'm looking into the specs to see if this possibility is mentioned..
If so, that is not supported by the IMS-Communicator...
The property <net.java.sip.communicator.sip.REGISTRAR_TRANSPORT/> was inherited from the original SIP-Communicator and the code that adds the transport=tcp parameter to the REGISTER's Request-URI was commented because it is mentioned in the 3GPP TS 24.229 that the REGISTER Request-URI must be equal to the domain name of the home network, and equal to the realm directive of the Authorization header. Do you agree?
 
But, if you want to use TCP, instead of UDP, for all requests and responses, you have to define the property <net.java.sip.communicator.sip.TRANSPORT value="TCP"/> and also change the transport parameter (to /tcp) in the property <javax.sip.OUTBOUND_PROXY/>.
 
About the second question, the Security Agreement is not yet fully implemented, but we are on our way.
