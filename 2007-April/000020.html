Hi Kiran,
 
«Caused by: javax.sip.SipException: Cold not create message channel - java.net.UnknownHostException: pcscf.mkd149633.wipro.com <http://pcscf.mkd149633.wipro.com/> »
 
can you ping "pcscf.mkd149633.wipro.com"?
 
As you can read in the output, the Exception retrieves the message "java.net.UnknownHostException: pcscf.mkd149633.wipro.com".
This means that the FQDN "pcscf.mkd149633.wipro.com" cannot be resolved by the DNS configured at your client side, making the jain-sip stack to fail when trying to acknowledge the call.
 
In the machine where you are using the IMS-Comm, configure a DNS that resolves "pcscf.mkd149633.wipro.com".
If you are in Windows, you can add a rule in the file %windir%\system32\drivers\etc\hosts (and manually insert the corresponding address for "pcscf.mkd149633.wipro.com")
In Linux, insert the DNS address which resolvs "pcscf.mkd149633.wipro.com" in the /etc/resolv.conf file.
