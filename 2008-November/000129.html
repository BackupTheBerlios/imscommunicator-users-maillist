> But, the SUBSCRIBE request with registration event was not getting routed.

> REGISTER and INVITE requests to the AS with appropriate trigger point

This is because the SUBSCRIBE with registration event is consumed by S-CSCF
and not routed further. All other SUBSCRIBE requests can be triggered on. 
Trigger points are only checked for requests routed through the S-CSCF and
for REGISTER requests (which is the only exception to allow 3rd an AS to be
informed about a registration). 

Why do you need this trigger?
Are you monitoring the P-CSCF with your AS?
