An issue that appears when we set <ACCEPT_REFER_WITHOUT_SESSION_ON
value="false"> and send a REFER message to the Communicator out of context
(that is, without sending before an INVITE), is that Communicator simply does
not respond to the query (so that we get a Request Timeout from IMScore after
some retries), where we would expect an answer like 406 or 488. What 
does come,
is a debugging message in the console ("Llegas aquí porque en esta nueva
transacción aún no hay INVITE a menos que sea de una sesión establecida
antes"). We use release 070427.
